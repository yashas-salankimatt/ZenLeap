### Separation of Concerns

The AI integration is a **completely separate extension** from ZenLeap. It ships as its own `.uc.js` file (`zenleap_agent.uc.js`) that gets loaded independently by fx-autoconfig alongside `zenleap.uc.js`. The two extensions:

- Share the same browser environment (chrome context, `gBrowser`, etc.)
- Can reference each other's globals if needed (e.g. `zenleap_agent.uc.js` can call ZenLeap APIs if ZenLeap exposes them on `window`)
- But **do NOT share source code** — no editing `zenleap.uc.js` to add agent features

If you need ZenLeap to expose something for the agent, note it in `notes.md` and flag it for the user — do not modify `zenleap.uc.js` yourself.

## Installation & Deployment

After ANY change to extension files:

1. Copy the AI integration files to the Zen Browser profile:
   ```bash
   PROFILE="$HOME/Library/Application Support/zen/Profiles/eumr3k85.Default (release)/chrome"

   # Main agent extension
   cp JS/zenleap_agent.uc.js "$PROFILE/JS/zenleap_agent.uc.js"

   # JSWindowActor files (create dir if needed)
   mkdir -p "$PROFILE/JS/actors"
   cp JS/actors/ZenLeapAgentParent.sys.mjs "$PROFILE/JS/actors/ZenLeapAgentParent.sys.mjs"
   cp JS/actors/ZenLeapAgentChild.sys.mjs "$PROFILE/JS/actors/ZenLeapAgentChild.sys.mjs"
   ```

2. **Verify every copy succeeded** — always diff after copying:
   ```bash
   diff JS/zenleap_agent.uc.js "$PROFILE/JS/zenleap_agent.uc.js"
   diff JS/actors/ZenLeapAgentParent.sys.mjs "$PROFILE/JS/actors/ZenLeapAgentParent.sys.mjs"
   diff JS/actors/ZenLeapAgentChild.sys.mjs "$PROFILE/JS/actors/ZenLeapAgentChild.sys.mjs"
   ```
   If any diff produces output, the copy failed — retry.

3. **Restart Zen Browser** for changes to take effect. You may quit and reopen Zen as many times as needed:
   ```bash
   osascript -e 'quit app "Zen"'
   sleep 2
   open -a "Zen"
   ```
   Wait a few seconds after reopen before interacting with the browser.

The target profile is **eumr3k85.Default (release)** — this is the profile with ZenLeap installed. Do NOT install to other profiles.

## Key Technical Pitfalls

- **Firefox strips `<button>` and `<input>` from innerHTML** — Always use `document.createElement()` + `appendChild()`.
- **`contentDocument` may be null** under Fission (process isolation) — Use JSWindowActors for cross-process content access.
- **`wrappedJSObject` bypasses Xray security** — Don't trust properties after unwrapping.
- **XPCOM actor module URIs** must be `resource:`, `chrome:`, or `file:` — not `data:` URIs.

## Zen Browser APIs Quick Reference

- **Tab management**: `gBrowser.addTab()`, `gBrowser.removeTab()`, `gBrowser.selectedTab`
- **Tab movement**: `gBrowser.moveTabBefore(tab, target)` / `gBrowser.moveTabAfter(tab, target)`
- **Workspaces**: `gZenWorkspaces.moveTabToWorkspace(tab, id)`, `gZenWorkspaces.changeWorkspaceWithID(id)`
- **Folders**: `gZenFolders.createFolder(tabs, { label, renameFolder })`
- **Screenshots**: `WindowGlobalParent.drawSnapshot()` (preferred), `PageThumbs.captureToBlob()` (fallback)
- **Content injection**: `Cu.exportFunction(fn, contentWindow, {defineAs: "name"})`
- **Actor registration**: `ChromeUtils.registerWindowActor()` or `ActorManagerParent.addJSWindowActors()`
- **TCP server**: `Cc["@mozilla.org/network/server-socket;1"].createInstance(Ci.nsIServerSocket)`
